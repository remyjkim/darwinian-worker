# v0.7.0 Release Readiness Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make the Bun-native `darwinian@0.7.0` release truthful and reproducible, integrate the standalone command bridge with independent verification, normalize the skills submodule checkout, and release from a verified clean commit.

**Architecture:** Keep the CLI's existing TypeScript/Bun npm entrypoint and declare Bun as a runtime prerequisite. Treat `drwn-command-bridge` as a nested standalone Node package with its own lockfile and verification command, while root CI installs and verifies it explicitly. Preserve the existing gitlink revision for `darwinian-worker-skills` and keep bridge publication gated on native macOS MCP validation.

**Tech Stack:** Bun 1.2, TypeScript, Bun test, npm pack/publish, Git submodules, GitHub Actions, MCP TypeScript SDK.
