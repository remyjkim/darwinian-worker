This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a bold aesthetic direction:
- **Purpose**: What problem does this interface solve? Who uses it?
- **Tone**: Pick an extreme: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian, and so on. Choose a direction that is true to the problem instead of following defaults.
- **Constraints**: Technical requirements such as framework, performance, and accessibility.
- **Differentiation**: What makes this interface memorable?

Choose a clear conceptual direction and execute it with precision. Bold maximalism and refined minimalism can both work; the key is intentionality, not intensity.

Then implement working code (HTML/CSS/JS, React, Vue, and similar) that is:
- production-grade and functional
- visually striking and memorable
- cohesive with a clear aesthetic point of view
- meticulously refined in every detail

## Frontend Aesthetics Guidelines

Focus on:
- **Typography**: Choose fonts that are distinctive and fit the concept. Avoid generic defaults like Arial, Inter, Roboto, and the system stack unless the existing design system requires them.
- **Color & Theme**: Commit to a cohesive palette. Use CSS variables for consistency. Strong dominant colors and deliberate accents beat timid, evenly distributed palettes.
- **Motion**: Use animation deliberately. Prefer a few meaningful moments such as page-load reveals, staggered entrances, or surprising hover states over constant micro-motion.
- **Spatial Composition**: Use asymmetry, overlap, density, or restraint intentionally. Avoid cookie-cutter layouts.
- **Backgrounds & Visual Details**: Add atmosphere with texture, gradients, patterns, borders, depth, or grain when it supports the concept.

Never fall back to generic AI aesthetics such as overused font families, purple-on-white defaults, predictable layouts, or interchangeable component patterns.

Interpret creatively and make unexpected choices that feel genuinely designed for the context. No design should converge on a single default look.

## Implementation Guidance

- Match implementation complexity to the aesthetic vision.
- Preserve the visual language of an existing product or design system when working inside one.
- Ensure the result works on both desktop and mobile.
- Use comments sparingly and only where the code would otherwise be difficult to read.

The goal is not decoration for its own sake. The goal is a clear, intentional interface that feels designed rather than assembled from defaults.
