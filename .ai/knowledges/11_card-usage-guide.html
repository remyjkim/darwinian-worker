drwn Cards and Workers

Author Cards → compose one Blueprint → add roots → select one Worker → write.

Terms

TermMeaning
CardAn independently authored and published capability bundle: skills, MCP definitions, hooks, and optional Mind content.
BlueprintA Card with an ordered list of plain Card members. It defines one composed Worker.
Worker rootAn installed plain Card or Blueprint. Installed roots are alternatives.
Selected WorkerThe single root named by activeWorker, or no root when it is null.
ClosureThe selected root followed by its ordered Blueprint members.

Author And Publish Cards

drwn card new @team/notion --no-git
drwn card source add-skill @team/notion notion-knowledge
drwn card source add-mcp @team/notion notion
drwn card source doctor @team/notion --json
drwn card publish @team/notion

Editable sources live under ~/.agents/drwn/sources. Published versions are immutable Store artifacts. Credentials never belong in a Card source or manifest.

Compose One Blueprint

drwn worker new @team/operator --no-git
drwn worker compose @team/operator --add @team/notion@^1.0.0
drwn worker compose @team/operator --add @team/fal@^1.0.0
drwn worker publish @team/operator

Member order is capability precedence and provenance order. Members must be plain Cards in this contract.

Initialize A Project

cd /path/to/project
drwn init --non-interactive
drwn apply @team/operator@^1.0.0
drwn use @team/operator --no-write
drwn write --dry-run
drwn write

The committed config is self-identifying:

{
  "schema": "drwn.project-config",
  "schemaVersion": 1,
  "workers": ["@team/operator@^1.0.0"],
  "activeWorker": "@team/operator"
}

workers lists only roots. Blueprint members are resolved into card.lock.

Manage Roots And Selection

drwn add @team/alternate@^1.0.0
drwn apply @team/operator@1.0.0 @team/alternate@1.0.0 --active @team/operator
drwn pin @team/operator@1.0.0
drwn update @team/operator
drwn remove @team/alternate
drwn use @team/operator
drwn use --none

Adding alternatives does not combine them. Composition changes happen by publishing a different Blueprint. Root mutations update config and lock atomically; selection changes do not alter the lock.

Lock And Generated State

PathRole
.agents/drwn/config.jsondrwn.project-config V1 intent: roots, one selection, and explicit overlays.
.agents/drwn/card.lockdrwn.project-lock V1 immutable root graph and Card provenance.
.agents/drwn/config.local.jsondrwn.project-local V1 ignored development overrides.
.agents/drwn/generated/workers.jsonDisposable registry of one aggregate generated bundle per root.
.agents/drwn/generated/active-worker.jsonDisposable selected root and closure record.
.agents/drwn/write-record.jsonOwnership hashes for safe reconciliation.

A Blueprint root receives one aggregate generated directory. Its member Cards do not receive sibling Worker directories unless they were separately installed as roots.

Project Capabilities

Project declarations come from the selected closure and explicit project overlays. Machine defaults are a separate machine-write authority. User-home skills and MCP entries may still be visible to downstream tools; status and doctor report those as ambient, never as project declarations.

drwn status --json
drwn status --why skill:notion-knowledge
drwn doctor --json

Notion And External Tools

Cards can ship a Notion MCP definition or an ntn integration skill, but OAuth and API keys remain operator state. Complete Notion OAuth in the downstream client and keep API keys in environment or secret storage. External stdio tools such as Momentic must be installed and authenticated separately.

An OAuth-required or missing-executable startup error is a runtime readiness issue. It does not mean the project root graph is invalid.

Clone And Install

drwn install --frozen
drwn install --no-write
drwn write --dry-run
drwn install

Install hydrates the exact lock and writes by default. It does not change root requirements or selection.

Capture And Deploy

drwn card new @team/captured --from-project .
drwn worker deploy @team/operator@1.0.0 --name team-operator

Capture includes the selected closure and explicit overlays, excluding inactive roots, machine/ambient capabilities, generated output, platform connectors, and secrets. Project-aware deploy sends one root plus pinned closure through the unchanged remote contract. Member and inactive-root deploy attempts fail locally.

Safety

drwn write is a pure projection and supports --dry-run. It never changes project root requirements, selection, or lock state.

Whole-Store export remains disabled with STORE_EXPORT_DISABLED_UNSAFE. Use the controlled reset runbook for unsupported development project state; there is no automated migration.

Future Machine Profile

The Recommended Darwinian Operator profile is an immutable @darwinian/operator@1.0.2 machine capability pin selected by opt-out guided setup. It projects 17 approved machine-safe skills and zero MCP servers, and it never creates Worker identity, hooks, permissions, governance, or project intent.
